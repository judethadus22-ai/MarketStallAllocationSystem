## 📌 Project Overview
The **Market Stall Allocation System** is designed to modernize and optimize the administration of public markets. By maintaining digitized records of traders, stalls, inspection schedules, and allocation terms, the system minimizes data redundancy and safeguards transactional integrity. 

This initiative supports **UN Sustainable Development Goal (SDG) 11: Sustainable Cities and Communities** by driving effective structural administration and boosting microeconomic recordkeeping efficiency.

## ⚙️ Features
*   **Comprehensive Relational Schema:** Includes dedicated tables for `Trader`, `Stall`, `Allocation`, `Inspector`, and `InspectionRecords`.
*   **Built-in Data Integrity:** Leverages Primary Keys, cascading Foreign Keys, strict `ENUM` definitions (for gender profiles and maintenance statuses), and unique identifier tags (`NationalID`, `StallNumber`).
*   **Audit-Ready Architecture:** Tracks physical conditions via structured inspection notes and localized timelines.
*   **Role-Based Security Control:** Fully configured user privilege limits directly execution-managed via MySQL.
